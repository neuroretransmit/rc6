Header only C++ implementation of the \href{doc/papers/RC6_Block_Cipher_586cc5d356330aef8a868aaa6c9bee493796.pdf}{\texttt{ R\+C6 block cipher}} implementation in \href{doc/papers/rfc8452.pdf}{\texttt{ G\+C\+M-\/\+S\+IV operating mode}}). Fully templated to accomodate different word sizes.\hypertarget{md__home_jonesy_git_rc6_README_autotoc_md1}{}\doxysection{Background}\label{md__home_jonesy_git_rc6_README_autotoc_md1}
\mbox{\hyperlink{classRC6}{R\+C6}} was an A\+ES candidate finalist that was \href{https://en.wikipedia.org/wiki/RC6\#Possible_use_in_NSA_\%22implants\%22}{\texttt{ found being used in N\+SA implants}}. The algorithm was an \href{https://patents.google.com/patent/US5835600A/en}{\texttt{ R\+SA patent}} but the patent expired between 2015 and 2017.

G\+C\+M-\/\+S\+IV was picked as a mode of operation for the following reasons.


\begin{DoxyEnumerate}
\item Nonce misuse resistant mode, the same nonce will not produce the same message -\/ the only information leaked is whether the two messages were equal or not.
\item Encrypted data is authenticated using additional data.
\item Parallelizable.
\item Can be implemented using A\+ES instructions (according to paper).
\end{DoxyEnumerate}\hypertarget{md__home_jonesy_git_rc6_README_autotoc_md2}{}\doxysection{Features}\label{md__home_jonesy_git_rc6_README_autotoc_md2}

\begin{DoxyItemize}
\item G\+C\+M-\/\+S\+IV mode of operation
\begin{DoxyItemize}
\item Authenticated Encryption with Additional Data
\item Nonce misuse resistant
\end{DoxyItemize}
\item Key size up to 2040 bits
\item Fully parameterized to support a variety of word lengths, key sizes and number of rounds.
